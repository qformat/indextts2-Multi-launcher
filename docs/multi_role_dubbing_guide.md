# IndexTTS 多角色配音与播客生成功能指南

本文档详细介绍了 IndexTTS Manager 的多角色配音（播客生成）功能。该功能允许用户通过简单的剧本格式，快速生成包含多角色对话、旁白、背景音乐和音效的完整音频内容。

## 1. 剧本解析规则 (Script Parsing)

IndexTTS 采用直观的“角色: 文本”格式来识别对话。

### 1.1 基本格式
每行代表一句对白，格式为 `角色名: 对白内容`。冒号支持中文 `：` 或英文 `:`。

```text
角色A: 你好，请问这里是 IndexTTS 的演示现场吗？
角色B: 是的，欢迎体验多角色配音功能。
(旁白) 两人相视一笑，走进了录音棚。
```

### 1.2 角色自动识别
系统会自动提取剧本中出现的所有角色标签。
*   **显式角色**: 行首匹配 `Name:` 或 `Name：` 的部分被视为角色名。
*   **旁白识别**:
    *   以 `(` 或 `（` 开头的内容，如 `(旁白)`、`（解说）`。
    *   不包含冒号的段落，默认可配置为归属到特定角色（如“默认A”或“旁白”）。
*   **未标注内容**: 对于没有明确角色标注的行，系统提供了多种分配模式：
    *   `默认A/B/C/D`: 归属于指定角色。
    *   `交替AB`: 自动在 A 和 B 之间交替分配。
    *   `ABC交替` / `ABCD交替`: 依次轮询分配。

## 2. 角色编排与音色分配 (Role Configuration)

解析剧本后，用户需要在 UI 中为提取出的每个角色配置音色。

### 2.1 独立音色配置
界面提供了 A、B、C、D 四个主要角色的配置槽位（对于更多角色，可在“批量编辑”中精细调整）。
*   **音色选择**: 支持从 `yinse/` 库中选择任意音色文件。
*   **筛选与搜索**: 可按分类筛选或直接搜索音色名。

### 2.2 随机音色 (Random Voice)
为实现“千人千面”的效果，系统支持**随机音色**功能。
*   **配置方式**: 在音色下拉框中选择 `🎲 随机选择 (当前分类)`。
*   **工作原理**:
    *   每次点击“生成”时，系统会从当前选定的分类文件夹中，随机抽取一个音色文件分配给该角色。
    *   **一致性**: 在同一次生成任务中，该角色的音色保持不变（不会出现第一句是男声，第二句变女声的情况）。
    *   **批量差异性**: 在批量处理多个脚本时，每个脚本的随机结果都是独立的。

## 3. 智能编辑器 (Batch Editor)

点击“编辑详情”按钮，进入可视化的表格编辑器，对每一行字幕进行精细控制。

### 3.1 行级控制
编辑器以表格形式展示所有对白，每行包含：
*   **文本**: 可直接修改对白内容。
*   **角色**: 修正解析错误，或临时将某句话指派给其他角色。
*   **音色预览**: 显示当前角色对应的音色文件名。
*   **语速 (Speed)**: 独立调整该句的语速（0.1x - 2.0x）。
*   **情感向量 (Emotion)**: 点击“情感向量”按钮，展开 8 维情感滑块（喜、怒、哀、惧等），为该句对白注入特定的情感色彩。

### 3.2 批量操作
*   **批量情感应用**: 在编辑器顶部设置一组情感向量，点击“应用到全部”或“应用到空行”，快速统一基调。
*   **批量角色调整**: 快速将所有未分配的行指定给某个角色。
*   **行操作**: 支持插入新行、删除行、上移/下移、合并行（回车向上合并）。

## 4. 音频后期编排 (Post-Processing)

系统不仅生成语音，还负责将多段语音拼接成完整的作品。

### 4.1 智能停顿 (Pacing)
为了让对话更自然，系统会根据标点符号自动插入静音片段：
*   **逗号/顿号**: 短停顿（如 200ms）。
*   **句号/问号/感叹号**: 中停顿（如 500ms）。
*   **段落/省略号**: 长停顿（如 800ms）。

### 4.2 背景音乐与闪避 (BGM & Ducking)
*   **BGM 叠加**: 支持上传背景音乐文件，自动循环铺满全长。
*   **自动闪避 (Auto-Ducking)**:
    *   **原理**: 检测人声活动。当有人说话时，BGM 自动降低音量（如降低到 30%）；当人声停止（停顿或对话间隙）时，BGM 自动恢复音量。
    *   **效果**: 模拟电台广播的专业混音效果，确保人声清晰且氛围感强。

## 5. 字幕生成 (Subtitle Generation)

*   **同步生成**: 随音频一同生成同名的 `.srt` 字幕文件。
*   **时间轴校准**: 字幕时间轴严格基于生成的音频长度计算，包含停顿时间，确保声画同步。
*   **格式标准**: 符合标准 SRT 规范，可直接导入剪映、Pr 等剪辑软件。

## 6. 批量播客生成 (Batch Podcast)

在“批量文档模式”下，支持一次性导入多个 `.txt` 剧本文件。

*   **队列处理**: 自动排队生成，无需人工值守。
*   **独立随机**: 如果配置了随机音色，每个剧本都会获得一组全新的角色音色组合。
*   **文件命名**: 自动清理输出文件名（去除时间戳后缀），直接使用剧本文件名命名 `.wav` 和 `.srt`，便于归档。
